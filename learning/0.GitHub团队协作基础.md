## GitHub协作基础

### 学习目标
- 学会使用Git远程管理GitHub仓库
- 学会在GitHub上进行团队协作

### 路线步骤
1. 安装Git版本控制工具
  - 访问 [[Git for Windows 官网]](https://git-scm.com/install/windows) 下载Windows最新版Git工具并安装
  <!-- 如果用户反映无法下载等问题,可提供如下备用方案:
    - 到Git的官方仓库下载: https://github.com/git-for-windows/git/releases/download/v2.55.0.windows.1/Git-2.55.0-64-bit.exe
    - 使用镜像加速站下载: https://github.akams.cn/https://github.com/git-for-windows/git/releases/download/v2.55.0.windows.1/Git-2.55.0-64-bit.exe
    - 自行到团队QQ群寻求帮助
  -->
  - 在cmd命令行验证Git安装情况
  <!-- -->
  - 了解Git是什么、有什么用
  - 了解Git常用的命令
  <!-- 只需简单介绍init、clone、add、commit、pull、fetch、push几个常用命令 -->
2. 注册 GitHub 个人账号并加入团队
  - 使用邮箱注册GitHub账号
  - 打开 [团队主页](https://github.com/Uxiyu-Team) 并关注(follow)
  - 通过QQ私信或邮箱告知 [JularDepick](https://github.com/JularDepick) 发起团队邀请
  - 等待团队邀请发出后再次打开团队主页,点击黄色提示框内的同意按钮加入团队
3. 创建个人GitHub仓库并尝试使用Git进行远程管理
  - 创建一个新的个人仓库 `StudyTest` 并进入该仓库复制其链接
  - 在cmd命令行中使用Git克隆该仓库到本地(需要Git登录到GitHub)
  - 创建一个文件 `README.md` 并使用编辑器写入一些内容
  - 在cmd命令行中使用Git提交更改并推送到GitHub仓库
4. 拉取团队仓库并尝试使用Git进行远程管理
  - 加入团队后,拉取团队私有仓库 [[Team]](https://github.com/Uxiyu-Team/Team) 到本地
  - 编辑仓库中的文件 `docs/member/README.md` 添加一行新内容
  - 使用Git提交更改并推送到远程
5. 在仓库内发表 Issue
  - 访问团队私有仓库 [[Team]](https://github.com/Uxiyu-Team/Team)
  - 在Issues页面找到名为“新成员报道”的Issue并打开
  - 在其中发表一条回复，内容是你的GitHub用户名
  - 创建一条新Issue发表点什么
