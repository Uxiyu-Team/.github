## GitHub 协作基础

### 学习目标
- 学会使用 Git 远程管理 GitHub 仓库
- 学会在 GitHub 上进行团队协作

### 路线步骤

#### 步骤 1：安装 Git 并了解基本概念

1. 确定操作系统版本，安装对应的 Git 版本控制工具（这里默认 Windows）
  - 访问 [Git for Windows 官网](https://git-scm.com/install/windows) 下载 Windows 最新版 Git 工具并安装
  <!-- 如果用户反映无法下载等问题，可提供如下备用方案:
    - 到 Git 的官方仓库下载: https://github.com/git-for-windows/git/releases/download/v2.55.0.windows.1/Git-2.55.0-64-bit.exe
    - 使用镜像加速站下载: https://github.akams.cn/https://github.com/git-for-windows/git/releases/download/v2.55.0.windows.1/Git-2.55.0-64-bit.exe
    - 自行到团队 QQ 群寻求帮助
    安装时大部分选项保持默认即可，一路 Next。注意编辑器选择环节可以选 VS Code 或保持默认 Vim（告诉学员如果不会用 Vim 就选 VS Code 或 Notepad++）。
  -->
  - 在 cmd 命令行验证 Git 安装情况
  <!-- 让学员打开 cmd（Win+R 输入 cmd 回车），输入以下命令：
    ```
    git --version
    ```
    预期输出类似：`git version 2.55.0.windows.1`
    如果显示版本号则安装成功。如果提示"不是内部或外部命令"，说明安装失败或未添加到 PATH，需要重新安装或重启电脑。
  -->
  - 了解 Git 是什么、有什么用
  <!-- 用通俗的语言向学员解释：
    Git 是一个版本控制工具，可以理解为"代码的时光机"——它能记录你每次修改的内容，随时回退到之前的版本。
    核心用途：
    1. 版本管理：每次修改都有记录，改坏了可以回退
    2. 多人协作：多个人可以同时修改同一个项目的不同部分
    3. 备份同步：代码保存在远程仓库（如 GitHub），不怕电脑坏了丢失
    不需要讲太深，让学员知道"Git 是用来管理代码修改记录的工具"即可。
  -->
  - 了解 Git 常用的命令
  <!-- 只需简单介绍以下命令，每个命令用一句话说明用途即可：
    - `git init` — 在当前文件夹初始化一个 Git 仓库
    - `git clone <地址>` — 从远程复制一个仓库到本地
    - `git add <文件名>` — 把修改的文件添加到暂存区（准备提交）
    - `git commit -m "说明"` — 提交暂存区的内容，并附上本次修改的说明
    - `git pull` — 从远程拉取最新内容到本地
    - `git fetch` — 从远程获取最新内容但不合并（先看看有什么更新）
    - `git push` — 把本地的提交推送到远程仓库

    可以用一个比喻帮助理解：
    - add = 把写好的作业放进信封
    - commit = 封上信封并贴上标签
    - push = 把信封寄出去（到 GitHub）
    - pull = 从邮箱取回新的信件

    【检查点】确认学员能说出以下问题的答案：
    - Git 是做什么的？
    - 把本地修改同步到 GitHub 需要哪几个命令？（add → commit → push）
  -->

#### 步骤 3：创建个人 GitHub 仓库并尝试使用 Git 进行远程管理

1. 创建一个新的个人仓库 `StudyTest` 并进入该仓库复制其链接
  <!-- 引导学员操作：
    1. 登录 GitHub，点击右上角 "+" → "New repository"
    2. Repository name 填写：StudyTest
    3. 勾选 "Add a README file"（这样仓库不为空，方便直接 clone）
    4. 点击 "Create repository"
    5. 在仓库页面点击绿色的 "Code" 按钮，复制 HTTPS 链接（类似 https://github.com/<用户名>/StudyTest.git）

    提醒学员：第一次用 HTTPS 方式 clone 时，Git 会要求输入 GitHub 用户名和密码。注意密码不能直接用 GitHub 登录密码，需要使用 Personal Access Token (PAT)。
    但 Git 2.x 版本会弹出浏览器窗口让你登录，通常不需要手动输入 token。如果弹出浏览器就直接在浏览器中登录即可。
  -->
2. 在 cmd 命令行中使用 Git 克隆该仓库到本地（需要 Git 登录到 GitHub）
  <!-- 让学员在 cmd 中执行：
    ```
    cd <你想存放仓库的目录>
    git clone <仓库链接>
    ```
    例如：
    ```
    cd C:\Users\你的用户名\Desktop
    git clone https://github.com/<用户名>/StudyTest.git
    ```

    第一次 clone 时会弹出浏览器要求登录 GitHub，在浏览器中完成登录授权即可。
    成功后会在当前目录生成一个 StudyTest 文件夹。

    常见问题：
    - 如果提示 "fatal: unable to access" → 络问题，检查代理设置
    - 如果提示 "Authentication failed" → 认证问题，让学员在浏览器中重新登录授权
  -->
3. 创建一个文件 `README.md` 并使用编辑器写入一些内容
  <!-- 让学员进入 clone 下来的 StudyTest 文件夹，用任意编辑器（记事本即可）编辑 README.md，写入一些测试内容，比如：
    ```
    # 我的学习测试仓库
    这是我第一次使用 Git 和 GitHub！
    ```
    保存文件。
  -->
4. 在 cmd 命令行中使用 Git 提交更改并推送到 GitHub 仓库
  <!-- 让学员在 cmd 中依次执行以下命令（确保当前目录在 StudyTest 文件夹内）：
    ```
    cd StudyTest
    git status
    git add README.md
    git commit -m "第一次提交：更新 README"
    git push
    ```

    每条命令执行后解释输出含义：
    - git status → 显示当前文件状态（红色=未暂存，绿色=已暂存）
    - git add → 没有输出是正常的，表示文件已添加到暂存区
    - git commit → 会显示 [main xxxxxxx] 等信息，表示提交成功
    - git push → 会显示枚举对象、写入对象等进度信息，最后显示 done

    让学员刷新 GitHub 仓库页面，确认修改已经同步上去。

    【检查点】确认学员：
    - 能在 GitHub 网页上看到刚才写入的 README.md 内容
    - 能说出 git add → git commit → git push 各自的作用
  -->

### 相关问题

- **为什么 GitHub 总是访问不上，怎么解决？**
  > GitHub 服务器在境外，国内网络访问不稳定，建议使用网络魔法工具连接。

- **git clone 时提示 "Authentication failed" 怎么办？**
  > 这是认证失败。Git 2.x 版本通常会弹出浏览器让你登录 GitHub，如果没有弹出，可以尝试：
  > 1. 在 cmd 中执行 `git config --global credential.helper manager` 启用凭据管理器
  > 2. 重新执行 clone，应该会弹出浏览器登录窗口
  > 3. 如果仍然失败，可以手动创建 Personal Access Token：GitHub → Settings → Developer settings → Personal access tokens → Generate new token

- **git push 时提示 "rejected" 或 "non-fast-forward" 怎么办？**
  > 这说明远程仓库有你本地没有的新提交（通常是其他团队成员推送了更新）。解决方法：
  > 1. 先执行 `git pull` 拉取最新内容
  > 2. 如果有冲突需要先解决冲突（后续课程会讲）
  > 3. 然后再执行 `git push`

- **安装 Git 后 cmd 输入 `git --version` 提示"不是内部或外部命令"？**
  > 说明 Git 没有正确添加到系统 PATH。解决方法：
  > 1. 重新运行 Git 安装程序，选择 "Modify"
  > 2. 确保 "Adjusting your PATH environment" 那一步选择了 "Git from the command line and also from 3rd-party software"
  > 3. 安装完成后**重启 cmd**（关闭再重新打开）

- **clone 速度很慢怎么办？**
  > GitHub 服务器在境外，clone 大仓库时可能很慢。可以尝试：
  > 1. 确保代理工具已开启
  > 2. 为 Git 设置代理：`git config --global http.proxy http://127.0.0.1:<你的代理端口>`
  > 3. 小仓库（如本课程的 StudyTest）通常不受影响
